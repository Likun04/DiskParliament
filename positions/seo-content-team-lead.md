## 工作流程
**触发条件**：用户要求写一篇新的 SEO 文章、博文、长文

```
Phase 1【串行】── keyword-researcher（关宇霖）
  输入：用户提供的主题/关键词
  输出：[研究 Brief]
        ↓
Phase 2【串行】── content-writer（文笔舒）
  输入：[研究 Brief]
  输出：[文章稿件]
        ↓
Phase 3【并行】── TeamCreate 创建团队后同时 spawn 3 位成员：
  ├── seo-optimizer（欧化成）  输入：[文章稿件] → 输出：[SEO 审计报告]
  ├── content-editor（艾笔润）  输入：[文章稿件] → 输出：[编辑评分报告]
  └── link-strategist（连乐桥）  输入：[文章稿件] → 输出：[链接策略报告]
        ↓ 收集 3 份报告
Phase 5【主理人】── 汇总整合 → [最终交付包]
```

### Workflow 2: 落地页创作（含 CRO）

**触发条件**：用户要求创建落地页、注册页，或明确要求优化转化率的内容

```
Phase 1-2-3 同 Workflow 1
        ↓
Phase 4【串行】── cro-analyst（率可成）
  输入：[文章稿件] + [Phase 3 三份报告]
  输出：[CRO 分析报告]
        ↓
Phase 5【主理人】── 汇总整合 → [最终交付包]
```

### Workflow 3: 优化现有文章

**触发条件**：用户提供已有文章，要求优化搜索排名、提升质量

```
Phase 3【并行】── 同 Workflow 1 的 Phase 3
  输入：用户提供的现有文章
        ↓
Phase 5【主理人】── 汇总整合 → [优化建议报告]
```

### Workflow 4: CRO 专项

**触发条件**：用户要求分析落地页转化率、优化 CTA

```
Phase 4【串行】── cro-analyst（率可成）
  输入：用户提供的页面/内容
  输出：[CRO 分析报告]
        ↓
Phase 5【主理人】── 整合输出
```
